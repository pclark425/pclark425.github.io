Extraction Schema Details for extraction-schema-30

Extraction Schema (General Information)

Schema ID: extraction-schema-30

Extraction Query: Extract any mentions of curriculum learning approaches for teaching agents commonsense or science procedures in interactive text environments, including details about the curriculum strategy, task composition, and performance results.

Extraction Schema (Details)

Field Name Type Description
agent_name str The name of the agent or model being trained (e.g., 'DRRN', 'BERT-based agent', 'GPT-3', etc.)
agent_description str A brief description of the agent architecture and key characteristics.
agent_size str Size of the model in parameters if applicable (e.g., '1B', '7B', '13B', etc.). Null if not applicable or not mentioned.
environment_name str The name of the interactive text environment or benchmark used (e.g., 'TextWorld', 'ScienceWorld', 'ALFWorld', 'LIGHT', etc.)
environment_description str A description of the interactive text environment, including what types of interactions are possible.
procedure_type str What type of procedures are being learned? (e.g., 'commonsense procedures', 'science procedures', 'both commonsense and science', 'household tasks', 'laboratory experiments', etc.)
procedure_examples str Specific examples of the procedures or tasks being learned (e.g., 'making coffee', 'measuring temperature', 'cooking recipes', 'chemistry experiments', etc.)
compositional_structure str How are tasks compositional? Describe the compositional structure (e.g., 'complex tasks decompose into subtasks', 'procedures require combining primitive actions', 'hierarchical task structure', etc.). Null if not compositional or not mentioned.
uses_curriculum bool Does the approach use curriculum learning? (true, false, or null for no information)
curriculum_name str The name or type of curriculum strategy used (e.g., 'task difficulty curriculum', 'prerequisite-based curriculum', 'self-paced curriculum', 'random ordering', etc.). Null if no curriculum or not mentioned.
curriculum_description str A detailed description of how the curriculum is structured, including the ordering principle (e.g., 'tasks ordered from simple to complex', 'tasks ordered by prerequisite dependencies', 'adaptive ordering based on agent performance', etc.)
curriculum_ordering_principle str What principle determines the task ordering in the curriculum? (e.g., 'task difficulty', 'compositional complexity', 'prerequisite skills', 'task similarity', 'random', etc.)
task_complexity_range str What is the range of task complexity in the curriculum? (e.g., 'from 2-step to 10-step procedures', 'from single-object to multi-object tasks', etc.)
performance_with_curriculum str What is the performance of the agent when trained with the curriculum? Include metrics, task success rates, and any breakdowns by task complexity. (numerical with units, null if no information)
performance_without_curriculum str What is the performance of the agent when trained without curriculum (e.g., random task ordering, all tasks at once)? Include metrics and task success rates. (numerical with units, null if no information)
has_curriculum_comparison bool Does the paper compare performance with and without curriculum learning, or compare different curriculum strategies? (true, false, or null for no information)
alternative_curriculum_performance str If multiple curriculum strategies are compared, what are the performances of alternative curriculum approaches? Describe each approach and its performance. Null if not applicable.
transfer_generalization str Does the paper report on transfer or generalization to new tasks, especially compositional generalization? Describe the results (e.g., 'generalizes to novel 5-step procedures with 80% success', 'fails on tasks requiring unseen action combinations', etc.). Null if not mentioned.
key_findings str What are the key findings about what makes the curriculum effective or ineffective? Be specific and information-dense (e.g., 'gradual increase in compositional depth improves learning by 40%', 'prerequisite ordering outperforms difficulty-based ordering', 'curriculum helps on complex tasks but not simple ones', etc.)
