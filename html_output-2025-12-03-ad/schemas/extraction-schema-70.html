Extraction Schema Details for extraction-schema-70

Extraction Schema (General Information)

Schema ID: extraction-schema-70

Extraction Query: Extract any mentions of language model agents using memory to solve text games, including details about the memory architecture, performance comparisons, and what makes memory effective or ineffective.

Extraction Schema (Details)

Field Name Type Description
agent_name str The name of the language model agent that plays text games.
agent_description str A brief description of the agent architecture, including the base language model used (e.g., GPT-3, GPT-4, LLaMA, etc.).
base_model_size str The size of the base language model in parameters (e.g., 1B, 7B, 13B, 175B, etc.). Null if not specified.
game_benchmark_name str The name of the text game or benchmark used for evaluation (e.g., Jericho, TextWorld, ALFWorld, ScienceWorld, etc.).
game_description str A brief description of the text game or benchmark, including the type of tasks involved.
uses_memory bool Does the agent use any form of memory system? (true, false, or null for no information)
memory_type str What type of memory does the agent use? (e.g., episodic memory, semantic memory, working memory, external memory buffer, retrieval-augmented memory, graph-based memory, etc.). Null if no memory or not specified.
memory_structure str Describe the structure and organization of the memory (e.g., key-value store, graph, vector database, sequential buffer, hierarchical, etc.). Null if not specified.
memory_content str What information is stored in the memory? (e.g., past observations, action history, discovered objects, room descriptions, task goals, successful strategies, etc.). Null if not specified.
memory_capacity str What is the capacity or size of the memory? (e.g., last 10 observations, 1000 tokens, unlimited, etc.). Null if not specified.
memory_retrieval_strategy str How does the agent retrieve information from memory? (e.g., recency-based, relevance-based retrieval, semantic similarity, full context, etc.). Null if not specified.
memory_update_strategy str How and when is the memory updated? (e.g., after each action, periodically, when new information is discovered, etc.). Null if not specified.
memory_usage_purpose str How does the agent use memory during gameplay? (e.g., for planning, tracking game state, avoiding repeated actions, remembering successful strategies, etc.). Null if not specified.
performance_with_memory str What is the agent's performance when using memory? Include specific metrics with units (e.g., success rate %, average score, completion rate, etc.). Null if not reported.
performance_without_memory str What is the agent's performance without memory (baseline)? Include specific metrics with units. Null if not reported.
has_memory_ablation bool Does the paper report an ablation study comparing performance with and without memory? (true, false, or null)
memory_effectiveness_findings str What specific findings does the paper report about what makes memory effective or ineffective? (e.g., 'memory helps most in long-horizon tasks', 'retrieval quality is more important than capacity', etc.). Null if not discussed.
memory_limitations str What limitations or challenges with the memory system are reported? (e.g., 'memory becomes cluttered with irrelevant information', 'retrieval errors compound over time', etc.). Null if not discussed.
comparison_with_other_memory_types bool Does the paper compare multiple different memory architectures or strategies? (true, false, or null)
best_memory_configuration str If multiple memory configurations are compared, which performed best and why? Null if not discussed.
