Extraction Schema Details for extraction-schema-110

Extraction Schema (General Information)

Schema ID: extraction-schema-110

Extraction Query: Extract any mentions of language models using self-reflection, self-critique, or iterative generate-then-reflect methods to improve answer quality, including details of the methods, tasks, performance with and without reflection, and any evidence of answer quality improvement or limitations.

Extraction Schema (Details)

Field Name Type Description
model_name str The name of the language model being evaluated (e.g., GPT-3, GPT-4, PaLM, Llama-2, etc.).
model_description str A brief description of the language model, including size (parameters), architecture, or other relevant details.
reflection_method_name str The name or label of the self-reflection or iterative refinement method used (e.g., 'generate-then-reflect', 'self-critique', 'reflection', etc.).
reflection_method_description str A concise description of how the self-reflection or iterative refinement method works (e.g., how reflection is implemented, what steps are involved, whether a separate model is used, etc.).
num_iterations int The number of generate-reflect iterations used in the method (null if not specified or not applicable).
task_name str The name of the task or benchmark used to evaluate the method (e.g., GSM8K, MMLU, TruthfulQA, etc.).
task_description str A brief description of the task or benchmark (e.g., 'grade-school math word problems', 'multi-step reasoning', etc.).
performance_with_reflection str The performance of the model on the task when using self-reflection or iterative refinement (include numerical values and units if available).
performance_without_reflection str The performance of the model on the task without using self-reflection or iterative refinement (include numerical values and units if available).
has_performance_comparison bool Does the paper report a direct comparison of performance with and without self-reflection or iterative refinement? (true, false, or null if no information)
evidence_of_improvement str A brief summary of the evidence (quantitative or qualitative) that self-reflection or iterative refinement improves answer quality.
limitations_or_failure_cases str Any reported limitations, failure cases, or situations where self-reflection or iterative refinement does not improve or may harm answer quality.
