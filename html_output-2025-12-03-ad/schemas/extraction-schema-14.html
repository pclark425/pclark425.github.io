Extraction Schema Details for extraction-schema-14

Extraction Schema (General Information)

Schema ID: extraction-schema-14

Extraction Query: Extract any mentions of how language models perform arithmetic operations, including the types of arithmetic tasks, model properties, performance results, methods used, and any mechanistic insights about how the models solve arithmetic problems.

Extraction Schema (Details)

Field Name Type Description
model_name str The name of the language model being evaluated (e.g., 'GPT-3', 'LLaMA-2', 'T5', etc.)
model_size str The size of the model in parameters (e.g., '125M', '1.3B', '7B', '175B', etc.). Null if not specified.
model_architecture str The architecture type of the model (e.g., 'decoder-only transformer', 'encoder-decoder', etc.). Null if not specified.
arithmetic_operation_type str The specific type(s) of arithmetic operation tested (e.g., 'addition', 'subtraction', 'multiplication', 'division', 'modular arithmetic', 'multi-step arithmetic', etc.). Be specific.
number_range_or_complexity str The range or complexity of numbers used in the arithmetic problems (e.g., 'single-digit', '2-digit addition', '3-digit multiplication', 'numbers up to 1000', etc.). Null if not specified.
method_or_intervention str Any special method, prompting strategy, or intervention used (e.g., 'zero-shot', 'few-shot', 'chain-of-thought prompting', 'scratchpad', 'fine-tuning on arithmetic', 'tool use', 'standard prompting', etc.)
performance_result str The performance/accuracy of the model on the arithmetic task. Be as specific as possible, including breakdowns by operation type, number of digits, or other relevant factors. Include units (e.g., '95% accuracy', '23% on 3-digit multiplication', etc.). Null if not reported.
mechanistic_insight str Any insights about HOW the model performs arithmetic - the mechanisms, representations, or processes involved (e.g., 'uses attention to track carry operations', 'memorizes frequent arithmetic facts', 'learns algorithmic solution', 'specific attention heads specialize in arithmetic', 'fails due to position encoding limitations', etc.). Null if no mechanistic insights provided.
performance_scaling str How performance scales with model size, training, or problem complexity (e.g., 'accuracy improves with model size', 'performance degrades with number of digits', 'emergent ability at 13B parameters', etc.). Null if not discussed.
failure_modes str Specific failure modes or error patterns observed (e.g., 'fails on problems requiring carries', 'makes off-by-one errors', 'struggles with larger numbers', 'confuses operation types', etc.). Null if not discussed.
comparison_baseline str If comparisons are made, what is being compared? (e.g., 'with vs without chain-of-thought', 'different model sizes', 'before vs after fine-tuning', 'different prompting methods', etc.). Null if no comparison.
key_finding str A concise, information-dense summary of the key finding(s) relevant to understanding how language models perform arithmetic. This should be punchy and highlight the most important takeaway.
