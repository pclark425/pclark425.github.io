Extraction Schema Details for extraction-schema-42

Extraction Schema (General Information)

Schema ID: extraction-schema-42

Extraction Query: Extract any mentions of actuator modeling, actuator dynamics, or motor modeling in the context of sim-to-real transfer for robotics, including details about what actuator properties were modeled, what was simplified, control frequencies, latencies, and whether transfer succeeded or failed.

Extraction Schema (Details)

Field Name Type Description
robot_platform str The name and type of robot platform used (e.g. 'Franka Panda manipulator', 'ANYmal quadruped', 'Shadow Hand', etc.)
actuator_type str The type of actuators used in the robot (e.g. 'electric DC motors with planetary gears', 'hydraulic actuators', 'pneumatic artificial muscles', 'series-elastic actuators', 'direct-drive motors', 'tendon-driven', 'cable-driven', etc.)
task_description str Brief description of the task being performed (e.g. 'bipedal locomotion', 'dexterous manipulation', 'high-speed pick-and-place', 'contact-rich assembly', etc.)
task_dynamics_category str Is the task quasi-static or dynamic? (e.g. 'quasi-static (motion time >5s)', 'dynamic (high-speed)', 'highly dynamic (impacts/jumps)', null if not specified)
control_mode str What control mode was used? (e.g. 'position control', 'torque control', 'impedance control', 'admittance control', 'operational-space control', 'hybrid position-force control', etc.)
control_frequency str What was the control frequency in Hz? (e.g. '50Hz', '500Hz', '1kHz', null if not reported)
actuator_properties_modeled str What specific actuator properties were explicitly modeled in simulation? List all that apply (e.g. 'torque limits', 'motor back-EMF', 'gear ratios', 'transmission efficiency', 'friction', 'backlash', 'control latency', 'motor inertia', 'electrical time constants', 'pressure dynamics', etc.)
actuator_properties_not_modeled str What actuator properties were NOT modeled or were simplified? (e.g. 'tendon mechanics simplified to direct joint control', 'no backlash modeling', 'no thermal effects', null if not specified)
control_latency_modeled str Was control latency/delay explicitly modeled? If yes, what was the latency value in milliseconds? (e.g. 'yes, 15ms', 'no', null if not specified)
actuator_parameter_randomization bool Were actuator parameters randomized during training? (true, false, or null)
randomization_parameters str If actuator parameters were randomized, which ones and by how much? (e.g. 'torque limits ±10%, motor strength ±20%, latency ±5ms', null if not applicable)
system_identification_used bool Was system identification used to identify actuator parameters from real robot data? (true, false, or null)
system_identification_method str If system identification was used, what method was used? (e.g. 'CMA-ES optimization', 'least-squares fitting', 'gradient-based optimization', 'Bayesian optimization', null if not applicable)
system_identification_parameters str If system identification was used, which actuator parameters were identified? (e.g. 'motor constants, friction coefficients, gear ratios', null if not applicable)
system_identification_sample_size str If system identification was used, how many real-world trajectories/episodes were needed? (e.g. '10 trajectories', '50 episodes', null if not reported)
transfer_success bool Did the sim-to-real transfer succeed? (true if successful, false if failed, null if unclear)
transfer_performance_quantitative str Quantitative transfer performance metrics (e.g. 'success rate: 85%', 'task completion time: 12.3s', 'tracking error: 0.05 rad', null if not reported)
ablation_study_actuator_modeling bool Does the paper include an ablation study comparing transfer performance with vs without actuator modeling? (true, false, or null)
ablation_results str If ablation study was performed, what were the results? (e.g. 'with actuator model: 90% success, without: 20% success', 'with latency modeling: stable, without: oscillations and failure', null if not applicable)
failure_mode_without_actuator_model str If transfer failed without proper actuator modeling, what was the specific failure mode? (e.g. 'joint oscillations', 'falls within 5 steps', 'overshooting by >50cm', 'instability', null if not applicable)
low_level_controller_present bool Is there a low-level controller (e.g. PD controller, impedance controller) between the policy and actuators? (true, false, or null)
low_level_controller_description str If a low-level controller is present, describe it (e.g. 'PD controller at 1kHz', 'impedance controller with variable stiffness', null if not applicable)
actuator_modeling_vs_contact_modeling str Does the paper discuss the relative importance of actuator modeling vs contact modeling? If so, what are the findings? (null if not discussed)
long_term_deployment bool Does the paper evaluate performance over long-term deployment (>10 hours)? (true, false, or null)
adaptation_or_wear_effects str If long-term deployment was studied, were actuator wear, temperature effects, or online adaptation discussed? (null if not applicable)
learned_vs_physics_actuator_model str Does the paper compare learned (neural network) actuator models vs physics-based models? If so, what are the results? (null if not discussed)
robot_size_scale str What is the approximate size/mass scale of the robot? (e.g. 'micro-robot <100g', 'desktop manipulator ~10kg', 'humanoid robot ~50kg', 'industrial robot >100kg', null if not specified)
key_findings_actuator_modeling str What are the key findings specifically about actuator modeling requirements for sim-to-real transfer? Be concise but information-dense, focusing on quantitative thresholds, necessary vs unnecessary modeling aspects, and conditions for success/failure.
