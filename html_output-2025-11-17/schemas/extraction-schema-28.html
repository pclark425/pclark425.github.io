Extraction Schema Details for extraction-schema-28

Extraction Schema (General Information)

Schema ID: extraction-schema-28

Extraction Query: Extract any mentions of navigation in text-based games or text worlds, including graph-topology features of the environments (such as diameter, clustering coefficient, dead-ends, door constraints, connectivity), exploration efficiency metrics, and how these relate to agent performance and policy structure.

Extraction Schema (Details)

Field Name Type Description
environment_name str The name of the text world, text game, or navigation environment being studied.
environment_description str A brief description of the text world or environment, including its domain (e.g., household navigation, fantasy game, etc.).
graph_diameter str The diameter of the environment's graph structure (maximum shortest path between any two nodes). Include numerical value and units if available, null otherwise.
clustering_coefficient str The clustering coefficient or measure of local connectivity in the environment graph. Include numerical value if available, null otherwise.
dead_ends_present bool Does the environment contain dead-ends (nodes with only one connection)? (true, false, or null for no information)
dead_ends_count str If dead-ends are present, how many or what proportion of the graph? Include numerical value if available, null otherwise.
door_constraints_present bool Does the environment have door constraints, locked doors, or conditional access requirements? (true, false, or null for no information)
door_constraints_description str If door constraints are present, describe their nature (e.g., 'requires key', 'one-way doors', 'conditional on previous actions').
graph_connectivity str Description of the graph connectivity or topology (e.g., 'fully connected', 'tree structure', 'sparse', 'dense', average degree, etc.).
environment_size str The size of the environment (e.g., number of rooms/nodes, number of connections/edges). Include numerical values if available.
agent_name str The name or type of agent/policy used for navigation (e.g., 'DQN', 'A3C', 'random walk', 'BFS', etc.).
agent_description str A brief description of the agent or policy, including key architectural features relevant to navigation.
exploration_efficiency_metric str What metric is used to measure exploration efficiency? (e.g., 'steps to goal', 'coverage rate', 'time to complete', 'sample efficiency').
exploration_efficiency_value str The measured exploration efficiency for this agent on this environment. Include numerical value and units if available, null otherwise.
success_rate str The success rate or completion rate of the agent on navigation tasks in this environment. Include numerical value (e.g., percentage) if available, null otherwise.
optimal_policy_type str Description of what type of policy is optimal or performs best in this environment (e.g., 'memory-based', 'reactive', 'planning-based', 'graph-search').
topology_performance_relationship str Describe any reported relationship between graph topology features and agent performance/exploration efficiency. Be specific about which topology features affect which performance metrics and how.
comparison_across_topologies bool Does the paper compare performance across different graph topologies or environment structures? (true, false, or null for no information)
topology_comparison_results str If comparisons across topologies are made, summarize the key findings about how different topological features affect navigation performance.
policy_structure_findings str Any findings about how optimal policy structure relates to environment topology (e.g., 'policies in high-diameter graphs require more memory', 'dead-ends require backtracking mechanisms').
