Extraction Schema Details for extraction-schema-13

Extraction Schema (General Information)

Schema ID: extraction-schema-13

Extraction Query: Extract any mentions of agents that use memory mechanisms to solve text-based games, including details about the memory architecture, the games played, and performance comparisons.

Extraction Schema (Details)

Field Name Type Description
agent_name str The name of the agent that plays text games (e.g., 'DRRN', 'CALM', 'ReAct', 'Reflexion', etc.)
agent_description str A detailed description of the agent architecture, including the base model/approach (e.g., 'LSTM-based RL agent', 'GPT-4 with prompting', 'transformer-based policy network')
agent_type str The category of agent (e.g., 'reinforcement learning', 'LLM-based', 'rule-based', 'hybrid', etc.)
uses_memory bool Does the agent use any form of memory mechanism? (true, false, or null for no information)
memory_type str What type of memory does the agent use? (e.g., 'episodic memory', 'semantic memory', 'working memory', 'external knowledge base', 'attention-based memory', 'graph-based memory', etc. null if no memory or no information)
memory_structure str Describe the structure and implementation of the memory (e.g., 'vector database of past observations', 'knowledge graph of entities and relations', 'buffer of recent actions and outcomes', etc. null if not applicable)
memory_content str What information is stored in the memory? (e.g., 'past observations and actions', 'object locations and properties', 'successful action sequences', 'world state representations', etc. null if not applicable)
memory_usage str How does the agent use memory during gameplay? (e.g., 'retrieves relevant past experiences for decision making', 'maintains world state model', 'recalls successful strategies', etc. null if not applicable)
game_benchmark str The name of the text game benchmark or environment (e.g., 'Zork', 'TextWorld', 'Jericho', 'ALFWorld', 'ScienceWorld', 'TWC', etc.)
game_description str Brief description of the game/benchmark, including key characteristics (e.g., 'procedurally generated cooking tasks', 'classic interactive fiction with complex puzzles', 'household tasks in simulated environments')
game_complexity str Indicators of game complexity such as average game length, number of objects, action space size, or other complexity metrics mentioned in the paper
performance_with_memory str Performance metrics when the agent uses memory (e.g., 'success rate: 45%', 'average score: 23.5', 'completion rate: 67%', include units and specific numbers. null if no information)
performance_without_memory str Performance metrics for a baseline/ablation without memory (e.g., 'success rate: 12%', 'average score: 8.3', include units and specific numbers. null if no information)
has_ablation_study bool Does the paper include an ablation study comparing performance with and without memory? (true, false, or null for no information)
memory_benefits str What specific benefits does memory provide? (e.g., 'better long-term planning', 'reduced redundant exploration', 'improved object tracking', 'faster learning', etc. null if not discussed)
memory_limitations str What limitations or failure cases are mentioned regarding the memory mechanism? (e.g., 'struggles with very long games', 'memory retrieval errors', 'scalability issues', etc. null if not discussed)
key_findings str Summarize the key findings about how memory helps (or doesn't help) in solving text games, in a concise and information-dense way
