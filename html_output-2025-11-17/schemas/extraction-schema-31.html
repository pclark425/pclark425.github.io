Extraction Schema Details for extraction-schema-31

Extraction Schema (General Information)

Schema ID: extraction-schema-31

Extraction Query: Extract any mentions of genetic/evolutionary algorithms that use crossover and mutation operations on code, programs, or literature to generate new solutions, with particular attention to measures of novelty, diversity, executability, and functionality.

Extraction Schema (Details)

Field Name Type Description
system_name str The name of the genetic/evolutionary system, algorithm, or method being described.
system_description str A detailed description of the genetic/evolutionary system, including its overall approach and goals.
input_type str What type of content is being evolved or mutated? (e.g. 'code', 'programs', 'literature', 'text and code', 'scientific papers', etc.)
crossover_operation str Describe the crossover operation used. How are two parent elements combined to create offspring? Be specific about the mechanism. (null if no crossover is used)
mutation_operation str Describe the mutation operation used. How are elements modified or perturbed? Be specific about the mechanism. (null if no mutation is used)
uses_literature bool Does the system use scientific literature, papers, or text as part of its genetic material? (true, false, or null for no information)
uses_code bool Does the system use code, programs, or executable blocks as part of its genetic material? (true, false, or null for no information)
novelty_metric str How is novelty, creativity, or originality measured in this system? Include the specific metric name and how it's calculated. (null if not measured)
novelty_results str What are the quantitative results for novelty/diversity? Include numbers and units. (null if no information)
executability_metric str How is executability, functionality, correctness, or validity measured in this system? Include the specific metric name and how it's calculated. (null if not measured)
executability_results str What are the quantitative results for executability/functionality? Include numbers and units. (null if no information)
diversity_metric str How is diversity of discovered solutions measured? (e.g. 'behavioral diversity', 'genotypic diversity', 'phenotypic diversity', etc.) (null if not measured)
diversity_results str What are the quantitative results for discovery diversity? Include numbers and units. (null if no information)
novelty_executability_tradeoff str Does the paper describe or demonstrate a tradeoff between novelty and executability? If so, describe the nature of this tradeoff with specific details and evidence. (null if not discussed)
frontier_characterization str Does the paper characterize a Pareto frontier or tradeoff curve between novelty and executability? If so, describe it quantitatively. (null if not discussed)
benchmark_or_domain str What benchmark, problem domain, or application area is the system evaluated on? (e.g. 'program synthesis', 'scientific discovery', 'code generation', etc.)
comparison_baseline str What baseline methods or systems is this approach compared against? List them briefly.
key_findings str What are the key findings or conclusions about how crossover/mutation affect novelty, executability, or diversity? Be specific and information-dense.
