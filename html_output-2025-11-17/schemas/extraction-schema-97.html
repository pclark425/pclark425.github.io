Extraction Schema Details for extraction-schema-97

Extraction Schema (General Information)

Schema ID: extraction-schema-97

Extraction Query: Extract any mentions of LLMs or AI systems being used to extract, discover, or distill quantitative laws, relationships, or patterns from scientific papers or literature.

Extraction Schema (Details)

Field Name Type Description
system_name str The name of the LLM system or method used to extract quantitative laws from papers.
system_description str A detailed description of how the system works, including the LLM architecture, prompting strategy, pipeline, or methodology used.
model_name str The specific LLM model used (e.g., GPT-4, Claude, LLaMA-2-70B, etc.). Null if not specified.
model_size str The size of the LLM model in parameters (e.g., 7B, 13B, 70B, 175B, etc.). Null if not specified.
scientific_domain str What scientific domain(s) were the papers from? (e.g., physics, chemistry, biology, materials science, computer science, etc.)
number_of_papers str How many papers were processed/analyzed by the system? Include the specific number or range if mentioned.
law_type str What type of quantitative law or relationship was extracted? (e.g., scaling laws, physical laws, empirical relationships, correlations, mathematical equations, etc.) Be specific.
law_examples str Specific examples of quantitative laws or relationships that were discovered/extracted by the system. Include the actual equations or relationships if provided.
extraction_method str How does the system extract quantitative information from papers? (e.g., parsing equations, extracting numerical data from tables/figures, text mining for relationships, etc.)
validation_approach str How were the extracted laws validated? (e.g., comparison with known laws, expert evaluation, empirical testing, cross-validation, etc.)
performance_metrics str What performance metrics were reported? Include accuracy, precision, recall, F1, or any domain-specific metrics with numerical values and units.
success_rate str What percentage or fraction of extracted laws were valid/correct? Include numerical values if reported.
challenges_limitations str What challenges, limitations, or failure modes were reported for the system?
comparison_baseline str Was the system compared to any baseline methods (human experts, traditional methods, other AI systems)? If so, what were the comparative results?
