Extraction Schema Details for extraction-schema-81

Extraction Schema (General Information)

Schema ID: extraction-schema-81

Extraction Query: Extract any mentions of language models using self-reflection or iterative generate-then-reflect methods to improve answer quality, including details of the method, tasks, and results.

Extraction Schema (Details)

Field Name Type Description
model_name str The name of the language model being evaluated (e.g., GPT-3, PaLM, Llama-2, etc.).
model_description str A brief description of the language model, including architecture, size, and any relevant training details.
reflection_method_name str The name or label of the self-reflection or iterative refinement method used (e.g., 'generate-then-reflect', 'self-consistency', 'self-critique', etc.).
reflection_method_description str A concise description of how the self-reflection or iterative process is implemented (e.g., prompting the model to critique its own answer, using multiple iterations, external feedback, etc.).
num_iterations int The number of generate-reflect iterations used in the method (null if not specified or not applicable).
task_name str The name of the task or benchmark on which the method is evaluated (e.g., GSM8K, MMLU, TruthfulQA, etc.).
task_description str A brief description of the task or benchmark.
performance_with_reflection str The performance of the model on the task when using self-reflection or iterative refinement (include metric and value, e.g., 'accuracy 85%').
performance_without_reflection str The performance of the model on the task without using self-reflection or iterative refinement (include metric and value, e.g., 'accuracy 78%').
has_performance_comparison bool Does the paper report a direct comparison of performance with and without self-reflection or iterative refinement? (true, false, or null if not specified)
qualitative_findings str Any qualitative findings about how self-reflection or iterative refinement affects answer quality (e.g., types of errors fixed, new failure modes, limitations, or notable examples).
counterexamples_or_negative_results str Any reported cases where self-reflection or iterative refinement did not improve or worsened answer quality, or failed to work as intended.
