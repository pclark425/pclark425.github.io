Extraction Schema Details for extraction-schema-71

Extraction Schema (General Information)

Schema ID: extraction-schema-71

Extraction Query: Extract any mentions of LLM-based agents that use memory systems to play text games, including details about the memory architecture, the text games played, and performance results.

Extraction Schema (Details)

Field Name Type Description
agent_name str The name of the LLM-based agent that plays text games.
agent_description str A detailed description of the agent architecture, including how it processes observations and generates actions.
base_llm str The base LLM used by the agent (e.g., GPT-3, GPT-4, LLaMA-7B, Claude, etc.), including model size if available.
uses_memory bool Does the agent use any form of memory system? (true, false, or null for no information)
memory_type str What type of memory does the agent use? (e.g., episodic memory, semantic memory, working memory, vector database, knowledge graph, retrieval-augmented, etc. null if no memory or no information)
memory_architecture str Detailed description of how the memory system is implemented and integrated with the LLM agent (e.g., how memories are stored, retrieved, and used).
memory_capacity str What is the capacity or size of the memory? (e.g., number of memories stored, context window size, etc. null if no information)
memory_retrieval_method str How does the agent retrieve relevant memories? (e.g., semantic similarity, recency, importance-based, learned retrieval, etc. null if no information)
memory_update_strategy str How and when are memories added or updated? (e.g., after each action, periodically, based on importance, etc. null if no information)
text_game_benchmark str The name of the text game or benchmark used for evaluation (e.g., Jericho, TextWorld, ALFWorld, ScienceWorld, etc.)
game_characteristics str Key characteristics of the text games used (e.g., puzzle complexity, world size, required reasoning types, episode length, etc.)
performance_with_memory str Performance metrics when the agent uses memory (e.g., success rate, score, completion rate, steps to completion). Include numerical values and units. null if no information.
performance_without_memory str Performance metrics for a baseline version without memory or with minimal memory. Include numerical values and units. null if no information.
has_ablation_study bool Does the paper include an ablation study comparing performance with and without memory or with different memory configurations? (true, false, or null)
memory_ablation_results str If ablation studies were conducted, what were the key findings about which memory components or configurations were most effective? null if no information.
comparison_with_other_memory_types str If the paper compares different memory approaches, what were the results? (e.g., 'episodic memory outperformed semantic memory by 15%'). null if no comparison.
key_findings_about_memory_effectiveness str What are the paper's main conclusions or findings about what makes memory effective for text games? (e.g., 'long-term memory crucial for puzzle games', 'retrieval quality more important than capacity', etc.)
