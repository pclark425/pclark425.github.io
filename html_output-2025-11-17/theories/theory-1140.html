Theory (General Information)

ID: theory-1140

Name: Dual-Process Reasoning Theory for Language Models

Type: general

Theory Query: Build a theory of how language models can best perform strict logical reasoning.

Description: This theory posits that language models (LMs) can best perform strict logical reasoning when they explicitly separate and coordinate two distinct reasoning processes: (1) a fast, pattern-based, intuitive process (System 1) and (2) a slow, rule-based, formal logical process (System 2). The LM must be architected or prompted to invoke System 2 for tasks requiring strict logic, and to cross-validate System 1 outputs with System 2, ensuring logical rigor and minimizing errors from pattern-matching alone.

Theory (Details)

Theory Statements

Statement 0: Dual-Process Activation Law (qualitative)

ConditionDetails
IF
• language model → receives → reasoning task
THEN
• language model → should activate → System 1 (pattern-based reasoning), and
• language model → should activate → System 2 (formal logical reasoning)

Supporting Evidence for this Law

  1. Human reasoning is often modeled as dual-process (Kahneman, 2011), and LMs show both intuitive and formal reasoning behaviors.
  2. LMs can generate both fast, plausible answers and slower, more accurate chain-of-thought outputs.

Existing Law Comparison

Likely Classification: somewhat-related-to-existing

Explanation: The law adapts dual-process theory to LMs and formalizes the need for explicit coordination for logical rigor.

What Already Exists: Dual-process theories are well-established in cognitive science; LMs have been observed to exhibit both intuitive and formal reasoning.

What is Novel: Explicitly requiring LMs to coordinate and cross-validate these processes for strict logic is new.

References:

  • Kahneman (2011) Thinking, Fast and Slow [dual-process theory in humans]
  • Wei et al. (2022) Chain-of-Thought Prompting Elicits Reasoning in Language Models [pattern vs. formal reasoning in LMs]

Statement 1: System 2 Override Law (qualitative)

ConditionDetails
IF
• reasoning task → requires → strict logical validity, and
• System 1 output → is inconsistent with → formal logic
THEN
• language model → should override → System 1 output with System 2 output

Supporting Evidence for this Law

  1. LMs often make pattern-based errors that can be corrected by explicit logical reasoning.
  2. Human studies show System 2 can override System 1 in logic tasks.

Existing Law Comparison

Likely Classification: somewhat-related-to-existing

Explanation: The law extends human dual-process override to LMs and strict logic.

What Already Exists: System 2 override is a known concept in human cognition.

What is Novel: Formalizing this as a requirement for LMs in strict logic tasks is new.

References:

  • Evans (2008) Dual-Processing Accounts of Reasoning, Judgment, and Social Cognition [System 2 override in humans]
  • Nye et al. (2021) Show Your Work: Scratchpads for Intermediate Computation with Language Models [LMs benefit from explicit reasoning steps]

New Predictions (Likely outcome)

  1. LMs with explicit dual-process coordination will outperform standard LMs on strict logic benchmarks.
  2. Pattern-based errors will decrease when System 2 is invoked for logic tasks.

New Predictions (Unknown outcome/high-entropy)

  1. Emergent meta-reasoning may arise from repeated System 1/System 2 cross-validation.
  2. Dual-process LMs may develop new forms of logical intuition not present in training data.

Negative Experiments

  1. If dual-process coordination does not improve logical accuracy, the theory is undermined.
  2. If System 2 fails to override System 1 errors, the sufficiency of the approach is questioned.

Unaccounted for Evidence

  1. Some LMs may lack the capacity to implement a true System 2 process due to architectural limitations.
