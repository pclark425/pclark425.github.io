Theory (General Information)

ID: theory-193

Name: System Identification for Sim-to-Real Transfer

Type: specific

Theory Query: Build a theory about sim-to-real transfer for scientific discovery agents that specifies environment fidelity requirements and skill-transfer conditions from virtual labs to real robotics labs, based on the following results.

Description: System identification—measuring or estimating simulator parameters from real-world data—can substantially reduce the sim-to-real gap when: (1) the simulator structure is correct but parameters are unknown, (2) a small number of key parameters (3-10) dominate the dynamics, and (3) sufficient real-world data (10-100 trajectories) can be collected safely. System identification is most effective when combined with domain randomization (to handle unidentified parameters) or online adaptation (to handle time-varying parameters). The effectiveness decreases when the simulator has structural errors or when parameters are highly correlated.

Theory (Details)

Theory Statements

  1. System identification is most effective when 3-10 key parameters dominate the dynamics and can be identified from 10-100 real trajectories
  2. Gradient-free optimization (CMA-ES, Differential Evolution) is effective for identifying 5-10 parameters from trajectory data
  3. Gradient-based optimization (differentiable simulation) is more sample-efficient but requires differentiable simulators and careful initialization
  4. Bayesian approaches (BayesSim) provide uncertainty estimates and can guide domain randomization, but require more computation
  5. System identification combined with domain randomization is more robust than either alone: identification handles known parameters, randomization handles unknown parameters
  6. Iterative identification (collect data, identify, retrain, repeat) can handle distribution shift as policies change
  7. System identification is less effective when simulator has structural errors (missing degrees of freedom, wrong contact model) rather than just parameter errors

Supporting Evidence

  1. System identification of gravity, mass, and damping via CMA-ES enabled peg-in-hole transfer without domain randomization [e1654.0] [e1654.2]
  2. Identification of cable stiffness and damping via inverse dynamics enabled flexible-object manipulation transfer [e1764.0]
  3. Identification of actuator parameters (DTE, friction) via black-box optimization enabled bipedal locomotion transfer [e1643.1] [e1643.2]
  4. SimOpt adaptation of simulator parameters using real rollouts enabled drawer-opening transfer [e1657.2]
  5. BayesSim posterior estimation over simulator parameters improved policy robustness for friction-sensitive tasks [e1785.1] [e1785.2]
  6. Simulator parameter tuning via Differential Evolution reduced trajectory errors for cable casting [e1679.0] [e1679.1] [e1679.2]
  7. TuneNet one-shot residual tuning estimated parameter differences from single observations [e1648.0]
  8. Grounded Action Transformation (GAT) learned action mappings to compensate for simulator-reality differences [e1795.0] [e1795.1]

New Predictions (Likely outcome)

  1. A manipulation task with 5 unknown parameters (mass, friction, damping) will achieve transfer with 20-50 real trajectories using CMA-ES identification
  2. A locomotion task will achieve 80% of expert performance after identifying 3-5 actuator parameters from 10-20 real episodes
  3. Combining system identification (for 5 key parameters) with domain randomization (for 10 secondary parameters) will outperform either alone
  4. Iterative identification with 3-5 iterations will converge to effective parameters for tasks where policy performance provides clear feedback

New Predictions (Unknown outcome/high-entropy)

  1. Whether learned system identification (neural networks, GPs) can replace optimization-based identification for transfer
  2. Whether system identification can handle time-varying parameters (wear, temperature) or only static parameters
  3. Whether there exists a minimum number of real trajectories required for identification that scales with parameter dimensionality
  4. Whether system identification can correct for structural model errors or only parametric errors

Negative Experiments

  1. Finding a task where system identification with 100+ trajectories fails to improve transfer would challenge the sample-efficiency claim
  2. Demonstrating successful identification of >20 parameters from limited data would challenge the dimensionality limit
  3. Showing that system identification without domain randomization outperforms combined approaches would challenge the complementarity principle
  4. Finding that identification converges to incorrect parameters but still achieves good performance would challenge the parameter-accuracy requirement

Unaccounted for Evidence

  1. How to determine which parameters to identify when the parameter space is large [e1654.0] [e1764.0]
  2. Whether identification effectiveness depends on the quality of the initial simulator or works equally well with poor initial models [e1648.0] [e1795.0]
  3. How identification requirements change with task complexity and policy expressiveness [e1657.2] [e1785.1]
