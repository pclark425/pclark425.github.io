Theory (General Information)

ID: theory-103

Name: Shortest-Path Planning with Learned Heuristics Theory

Type: specific

Theory Query: Build a theory of how agents perform planning with external tools in partially observable text environments, including belief-state updates that incorporate tool outputs and guide shortest-path actions, based on the following results.

Description: In partially observable text environments with explicit spatial structure (rooms, locations, navigation graphs), agents that combine explicit graph-based representations of connectivity with learned heuristics for navigation achieve more efficient goal-reaching than pure learned policies or pure classical planners alone. The graph representation provides a structured substrate for planning that handles spatial connectivity, while learned components (value functions, neural scorers, or LLM reasoning) handle partial observability, uncertain state estimation, and action selection under ambiguity. The benefit of this hybrid approach is most pronounced in environments with: (1) non-trivial spatial structure (multiple rooms, complex connectivity), (2) long-horizon navigation requirements, and (3) partial observability where classical heuristics (e.g., Euclidean distance) are unavailable or unreliable. However, the effectiveness critically depends on accurate graph construction, which in text-only environments must be inferred from descriptions.

Theory (Details)

Theory Statements

  1. Explicit graph representations of spatial connectivity enable more efficient navigation than purely learned policies in environments with non-trivial spatial structure (multiple rooms, complex connectivity patterns).
  2. Learned heuristics (value functions, neural network scorers, LLM reasoning) can guide graph search or action selection in partially observable settings where classical heuristics (e.g., Euclidean distance) are unavailable or unreliable due to incomplete state information.
  3. Hybrid approaches that combine graph-based path planning with learned action selection outperform either pure classical planning or pure learning in partially observable spatial environments, particularly for long-horizon navigation tasks.
  4. The effectiveness of graph-based navigation depends critically on the quality of graph construction: accurate connectivity information is essential, and errors in graph construction can negate benefits.
  5. In text-only environments, spatial graphs must be inferred from textual descriptions, making graph construction itself a challenging learning problem that requires entity extraction, relation detection, and temporal integration.
  6. Graph-based navigation provides diminishing returns in environments where: (1) spatial structure is trivial (single room, linear paths), (2) navigation is not the primary challenge, or (3) the environment is fully observable and deterministic.
  7. Classical shortest-path algorithms (A*, Dijkstra) can be effectively combined with learned components by using learned heuristics for node scoring, learned graph construction, or learned action execution.

Supporting Evidence

  1. NAIL uses knowledge graph to track locations and connections, uses graph-based navigation decisions, achieving 2.56% completion with graph-based routing to promising nodes [e883.3]
  2. Golovin maintains explicit map graph of locations and movement commands, uses graph-based routing to promising destinations with distance heuristics, achieving 1.45% completion [e883.1]
  3. LLM+P with Fast-Downward classical planner achieves 90% success on BlocksWorld using numeric distance fluents and total-cost metric for navigation optimization, demonstrating effectiveness of symbolic planning with numeric heuristics [e882.0]
  4. ConceptAgent uses ROS2 Nav2 with real-time point clouds for waypoint navigation, combining classical path planning (Nav2 global/local planners) with LLM-guided high-level planning [e804.0]
  5. TextWorld engine tracks agent orientation and computes relative directions (RD = (AO − DN) mod 8) for egocentric navigation, enabling navigation commands without requiring agents to maintain full trajectory memory [e788.5]
  6. LLM agent with Navigation Module (map/route extractor) that returns textual route descriptions and next-step directions achieves ~88.7% average performance on symbolic tasks including MapReader [e770.0]
  7. NeuAgent learns to navigate Wikipedia graph using learned policy over hyperlinks, achieving Recall@1=36.7% by learning to select navigation actions in a graph structure [e784.0]
  8. KNOWNO uses waypoint navigation with ROS2 Nav2 and local costmaps for reactive obstacle avoidance, combining classical navigation with LLM planning [e876.0]
  9. Q*BERT uses knowledge graph to represent room connectivity with directional edges, uses graph to constrain navigation actions, though no explicit shortest-path algorithm is employed [e797.0]
  10. Inner Monologue uses scripted navigation policies with ground-truth map together with learned perception, combining classical navigation primitives with LLM high-level planning [e875.1]
  11. DiffG-RL uses Difference Graph representation but navigation is performed by learned policy selecting discrete move commands, not explicit path planning [e778.0]
  12. Ammanabrolu & Riedl graph-based RL uses knowledge graph to represent spatial structure and prune actions, demonstrating benefit of explicit spatial representation [e767.0] [e806.5]
  13. KG-A2C uses knowledge graph with room-to-room relations for navigation, learned policy issues movement actions constrained by KG [e797.3]
  14. GATA uses belief graph that can represent spatial connectivity, though navigation is via learned policy not explicit search [e794.1]

New Predictions (Likely outcome)

  1. An agent that uses A* search with learned heuristics (from a neural network trained on successful trajectories) will find shorter paths than an agent using only learned policy in grid-world-like text environments with 10+ rooms.
  2. In text-based games with explicit room descriptions and directional commands, agents that maintain explicit room-connectivity graphs will reach navigation goals in fewer steps than agents without graphs, even when both use learned action selection.
  3. Agents that combine classical path planning (A*, RRT) with learned perception modules for graph construction will outperform end-to-end learned navigation in environments with complex but structured spatial layouts.
  4. In environments where navigation is a bottleneck (e.g., must visit multiple distant locations), graph-based planning will show larger performance gains compared to environments where navigation is trivial.

New Predictions (Unknown outcome/high-entropy)

  1. Whether learned end-to-end navigation policies can match the sample efficiency of graph-based approaches when given sufficient training data (millions of episodes) and modern architectures (transformers with large context).
  2. Whether there exists a universal learned heuristic function that works across diverse spatial environments (indoor navigation, outdoor navigation, abstract graphs), or whether heuristics must be environment-specific or learned per-domain.
  3. Whether graph-based navigation remains beneficial in very high-dimensional or continuous spatial environments where discrete graphs are difficult to construct, or whether learned representations (e.g., neural maps) are more effective.
  4. Whether probabilistic graph representations (with uncertain edges and nodes) provide significant benefits over deterministic graphs in partially observable settings, and whether the computational cost is justified.
  5. Whether hybrid approaches that dynamically switch between graph-based planning and learned policies based on environment characteristics can achieve best-of-both-worlds performance.

Negative Experiments

  1. Demonstrating that pure learned policies (e.g., transformers with sufficient capacity and training data) achieve the same path efficiency and sample efficiency as graph-based planners in structured spatial environments would challenge the necessity of explicit graphs.
  2. Showing that random walk or simple heuristics (e.g., always go to unexplored rooms) perform as well as shortest-path planning in certain text-based environments would question the value of path optimization.
  3. Finding that graph-based navigation provides no benefit over learned policies in environments with clear spatial structure and long-horizon navigation requirements would challenge the theory's core claims.
  4. Demonstrating that the computational cost of maintaining and updating graphs outweighs the benefits in terms of sample efficiency or final performance would challenge the practical utility of the approach.
  5. Showing that errors in graph construction (from text) are so frequent that they negate any benefits of graph-based planning would challenge the feasibility of the approach in text-only environments.

Unaccounted for Evidence

  1. How to automatically determine when to use graph-based vs. learned navigation based on environment characteristics (spatial complexity, observability, task horizon)
  2. The computational costs of maintaining and updating spatial graphs in real-time, particularly for large environments or when graph construction is expensive
  3. How to handle environments with multiple valid path types (e.g., shortest vs. safest vs. most informative) and how to learn or specify the appropriate objective
  4. How to handle dynamic environments where spatial structure changes over time (doors open/close, new rooms appear) and graphs must be updated online
  5. The trade-offs between graph granularity (fine-grained vs. coarse-grained spatial representations) and planning efficiency vs. accuracy
  6. How to integrate uncertainty in graph construction (probabilistic edges, uncertain node locations) into planning algorithms
  7. Whether graph-based approaches scale to very large environments (hundreds or thousands of locations) or whether hierarchical representations are necessary
