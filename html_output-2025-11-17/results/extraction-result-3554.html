Hyperparameters

As training of the generative model takes a substantial amount of time (several days on our NVIDIA P100 GPU with 12GB of memory), an exhaustive grid search of hyper-parameters for training is prohibitive. For our experiments, we mainly adhere to the standard hyperparameters reported by Gebauer et al. ${ }^{1}$ which are available in their open source implementation of G-SchNet. We use the same optimizer, learning rate scheduling, and Gaussian expansion width for ground truth distances. Only a few parameters are adapted in order to meet the increased memory demands that stem from the significantly increased size of molecules compared to the structures previously used with G-SchNet. ${ }^{1}$ We limit the number of features in the atom-wise representations to 64 and make use of 6 interaction layers in the feature learning block. Furthermore, we randomly sample 5 atom placement steps per molecule in each epoch instead of using the whole trajectory and set the mini-batch size to 5 molecules, which means that exactly 25 atom placement steps are included in each batch. G-SchNet allows to tune the randomness of atom placements during generation with a temperature parameter in the 3d grid distribution. We stick with the reported grid parameters and also adhere to the optimal temperature value of 0.1 determined by Gebauer et al. ${ }^{1}$ using DFT reference calculations to validate the quality of generated structures.
